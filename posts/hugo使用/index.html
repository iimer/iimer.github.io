新建项目

hugo new site [项目名称]

进入项目根目录

# 本地启动
hugo server

新建文件

进入项目根目录

hugo new [文件名]

新文件默认为草稿，对应draft: true。

要想启动时显示该文件，需要加上参数-D或者将draft的值改为false。

hugo server -D

在创建完第一个文件后，可以在此文件目录下直接创建别的文件，不需要每次都敲代码。

但必须保证开头的格式与第一个文件相同。

引入主题

基本套路

  1. 下载对应github的zip包，并解压到项目themes目录下。

  2. 修改根目录下config.toml文件，指定对应的主题，如LoveIt。

    # 更改使用 Hugo 构建网站时使用的默认主题

theme = "LoveIt"

  3. 其余按照主题特定的文档修改即可。

分离项目

一般我们记录博客是不会直接操作public里面的静态文件的，所以存储hugo项目才是最为重要的事情。

分离主要是public和源hugo项目：

  1. public关联部署的库
  2. 源hugo关联新库，并用.gitignore文件来隔离public文件夹。

在我们写完博客后，要将以上两个项目分别提交。
